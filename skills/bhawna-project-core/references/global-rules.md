# Bhawna Foundation Project Rules

## Global rules

- Follow the Pencil design closely.
- Keep everything responsive on desktop, tablet, and mobile.
- Do not break previously completed work.
- Keep code modular, production-ready, and scalable.
- Use JavaScript only.
- Use Tailwind CSS for styling.
- Maintain a consistent design language across all pages.
- Infer missing states and components in the same style as the design.
- Build real working features, not fake placeholders.
- Keep the public site and admin panel responsive.
- Add validation, loading states, error states, empty states, and success states.
- Keep backend and frontend cleanly separated.
- Prefer reusable components over repetition.

## Phase rule

Only do the scope of the current phase. Do not jump ahead unless a small architectural decision is required to keep later phases clean.

## Phase closeout rule

At the end of each phase, report:

1. what was completed
2. files created or updated
3. remaining assumptions
4. anything that should be done in the next phase

## Phase planning hint

For this repo, safe early phases are usually:

1. project scaffolding and shared design system
2. public layout and one complete page at a time
3. content APIs and real form workflows
4. admin panel and content management
5. hardening, SEO, and release preparation
